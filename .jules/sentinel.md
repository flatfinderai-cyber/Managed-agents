## 2024-05-12 - Fix timing attack vulnerability in API key comparisons
**Vulnerability:** Admin API endpoints in `routes/human_review.py`, `routes/search_blitz.py`, `routes/orchestrator.py`, and `routes/stack_decisions.py` are verifying the `x-internal-key` header using a simple string equality check (`x_internal_key != expected`). This is vulnerable to a timing attack where an attacker could deduce the internal API key character-by-character based on the time it takes for the server to reject invalid keys.
**Learning:** Comparing security-sensitive strings like API keys or tokens with standard equality operators (`==`, `!=`) allows timing differences because string comparison returns early as soon as a character mismatch is found.
**Prevention:** Always use constant-time comparison functions like `secrets.compare_digest()` from the Python standard library to compare security-sensitive strings, which guarantees that the comparison time depends only on the length of the string and not the contents, preventing timing attacks.
