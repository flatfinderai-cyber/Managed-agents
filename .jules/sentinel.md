## 2025-02-14 - Fix API Key Timing Attack
**Vulnerability:** The internal API key (`INTERNAL_API_KEY`) was being verified using the standard string inequality operator (`!=`) against the `x-internal-key` header in multiple route files (`routes/human_review.py`, `routes/orchestrator.py`, `routes/search_blitz.py`, and `routes/stack_decisions.py`). This allows attackers to perform a timing attack by measuring the time it takes for the server to reject a request, thereby deducing the correct API key character by character.
**Learning:** Comparing security-sensitive strings using standard equality operators (`==` or `!=`) is vulnerable to timing attacks because the comparison short-circuits as soon as a mismatch is found.
**Prevention:** Always use constant-time comparison functions like `secrets.compare_digest()` from the `secrets` module when comparing sensitive strings such as API keys, tokens, or passwords to ensure that the time taken for comparison does not depend on the input values.
